# LPO（Linear-drive Pluggable Optics）線性驅動可插拔光學深度分析

> 研究日期：2026-02-14  
> 主題：LPO 技術原理、與 DSP/CPO 比較、市場趨勢與投資機會

---

## 📋 摘要

LPO（Linear-drive Pluggable Optics，線性驅動可插拔光學）是 AI 資料中心光通訊技術的重要演進，透過**移除光模組內的 DSP 晶片**，將訊號處理功能整合至交換晶片端，實現 30-50% 功耗降低、更低延遲與更低成本。作為介於傳統 DSP 光模組與 CPO 之間的過渡方案，LPO 在 **2025-2027 年**將成為 AI 資料中心短距離互連的主流選擇。

**關鍵洞察**：
- LPO 功耗比傳統 DSP 模組降低 **30-50%**
- 延遲降低約 **100ns**（移除 DSP 處理延遲）
- 成本降低 **20-40%**（簡化光模組設計）
- 適用距離：**<100-500m**（機房內互連）
- **OIF（Optical Internetworking Forum）已發布 LPO 白皮書**，標準化進展中
- 主要推動者：**Meta、Microsoft、Google、NVIDIA**

---

## 🔬 技術原理

### 什麼是 LPO？

傳統高速光模組（如 400G/800G QSFP-DD）內建 **DSP（數位訊號處理器）** 晶片，用於：
- 訊號等化（Equalization）
- 前向糾錯（FEC）
- 時脈資料回復（CDR）

LPO 的核心概念是：**將這些 DSP 功能從光模組移至交換晶片（Switch ASIC）**，光模組僅保留線性的光電轉換功能。

```
傳統 DSP 架構：
[Switch ASIC] ←SerDes→ [光模組 DSP + 光引擎] ←光纖→ [遠端]
               ↑
          兩端各有 DSP（功耗×2）

LPO 架構：
[Switch ASIC + DSP] ←線性訊號→ [光模組(純光電轉換)] ←光纖→ [遠端]
                 ↑
            DSP 整合在 ASIC，光模組簡化
```

### 核心技術元件

| 元件 | DSP 架構 | LPO 架構 | 差異 |
|------|----------|----------|------|
| **SerDes** | ASIC 內建 | ASIC 內建 | 相同 |
| **DSP/CDR** | 光模組內建 | ASIC 內建 | **移除** |
| **FEC** | 光模組處理 | ASIC 處理 | **移除** |
| **驅動器** | 數位驅動 | 線性驅動 | 簡化 |
| **TIA** | 傳統設計 | 線性設計 | 簡化 |
| **光引擎** | EML/VCSEL | EML/VCSEL | 相同 |

### LPO 的三種實現形式

根據 OIF 定義，LPO 有三種變體：

```
1. EML（External Laser Module）
   - 外部雷射光源
   - 最低功耗、最高整合度
   - 適合未來 CPO 演進

2. 線性收發器（Linear Transceiver）
   - 完整光模組，但無 DSP
   - 可直接取代傳統模組
   - 2025-2026 主流

3. 線性接收器（Linear Receiver）
   - 僅接收端採用線性設計
   - 發送端仍用 DSP
   - 過渡方案
```

## ⚡ LPO vs DSP vs CPO 比較

### 架構比較

| 項目 | DSP（傳統） | LPO | CPO |
|------|-------------|-----|-----|
| **光模組形式** | 可插拔 | 可插拔 | 共封裝 |
| **DSP 位置** | 光模組內 | Switch ASIC | Switch ASIC |
| **電路徑長度** | 長（數十公分） | 中（數十公分） | 短（毫米級） |
| **光模組功耗** | 高（含 DSP） | 低（無 DSP） | 最低 |
| **系統功耗** | 基準 | -30~50% | -50%+ |
| **延遲** | 高（+DSP 處理） | 中（-100ns） | 最低 |
| **成本** | 基準 | -20~40% | 初期較高 |
| **可維護性** | ✅ 可熱插拔 | ✅ 可熱插拔 | ❌ 需換板 |
| **適用距離** | 長距離 OK | <500m 最佳 | <100m 最佳 |
| **標準化** | 成熟 | 發展中 | 發展中 |
| **商用時程** | 現有 | 2025-2026 | 2026-2027 |

### 功耗比較（800G 模組）

| 架構 | 每通道功耗 | 總模組功耗 | 備註 |
|------|-----------|-----------|------|
| DSP OSFP | ~2.5W/lane | 20-25W | 含內建 DSP |
| LPO OSFP | ~1.5W/lane | 12-15W | 無 DSP |
| CPO | ~0.8W/lane | 6-8W | 最短電路徑 |

### 延遲比較

| 架構 | SerDes→光纖延遲 | 備註 |
|------|----------------|------|
| DSP | ~200-300ns | DSP 編解碼 + CDR |
| LPO | ~100-150ns | 移除 DSP 處理 |
| CPO | <50ns | 最短路徑 |

## 📈 市場驅動因素

### 1. AI 資料中心功耗危機

AI 訓練叢集的功耗問題已達臨界：

| 系統 | GPU 功耗 | 網路/光模組功耗 | 總功耗 |
|------|---------|----------------|--------|
| DGX H100 (8 GPU) | 5.6 kW | 1.2 kW | 10.2 kW |
| DGX B200 (8 GPU) | 8.0 kW | 1.8 kW | 14+ kW |
| GB200 NVL72 | 120 kW+ | 20 kW+ | 140+ kW |

LPO 可將網路/光模組功耗降低 30-50%，對於百萬 GPU 級叢集，每年可節省數百萬美元電費。

### 2. 延遲敏感的 AI 工作負載

AI 訓練中的 All-Reduce 通訊對延遲極度敏感：
- 每 100ns 延遲增加 → 整體訓練時間增加 0.1-0.5%
- 百萬 GPU 叢集每降低 100ns → **年省數週訓練時間**
- LPO 比 DSP 節省約 100ns 延遲

### 3. 超大規模部署成本

| 部署規模 | 光模組數量 | 成本差異（DSP vs LPO） |
|----------|-----------|----------------------|
| 10,000 GPU 叢集 | ~80,000 | $8-12M 節省 |
| 100,000 GPU 叢集 | ~800,000 | $80-120M 節省 |

### 4. 可維護性需求

與 CPO 相比，LPO 保留了可插拔特性：
- 光模組故障可現場更換（5 分鐘）
- CPO 故障需更換整塊主板（數小時 + 系統重啟）
- 對於 SLA 要求高的企業客戶，可維護性是關鍵

## 🔧 技術挑戰與解決方案

### 1. 線性通道損耗補償

**挑戰**：移除 DSP 後，無法用數位方式補償通道損耗  
**解決方案**：
- 限制傳輸距離（<500m）
- 使用高品質低損耗光纖
- ASIC 端整合更強的等化器

### 2. 互通性（Interoperability）

**挑戰**：不同廠商的 LPO 模組是否能互通？  
**解決方案**：
- OIF 正在制定 CEI-112G-LR（Linear Receiver）規範
- IEEE 802.3 考慮納入 LPO 標準
- 超大規模業者主導規格（實質標準）

### 3. 長距離限制

**挑戰**：LPO 在長距離（>500m）表現不如 DSP  
**解決方案**：
- LPO 定位於機房內（<100-500m）互連
- 長距離 DCI（Data Center Interconnect）仍用 DSP 或相干光學
- 分層架構：短距 LPO + 長距 DSP/Coherent

### 4. 散熱與熱可靠性

**優勢**：LPO 功耗低，散熱需求降低  
**注意**：但需確保線性元件在溫度變化下的穩定性

### 與 CPO 的互補關係

```
2024-2026：DSP 主導（80%），LPO 萌芽（15%），CPO 試驗（5%）
2027-2028：LPO 崛起（40%），DSP 下降（45%），CPO 起飛（15%）
2029-2030：CPO 主流化（30%），LPO 成熟（40%），DSP 利基（30%）
```

**結論**：LPO 是 DSP→CPO 的**橋樑技術**，在 2025-2028 年將有顯著市場份額。

## 📝 結論

LPO 是 AI 資料中心光通訊演進的重要里程碑：

1. **技術定位**：介於 DSP（高功耗/高成本）與 CPO（高整合/不可維護）之間
2. **核心價值**：30-50% 功耗降低、20-40% 成本降低、~100ns 延遲改善
3. **市場窗口**：2025-2028 年黃金期，之後逐步被 CPO 取代
4. **採用路徑**：超大規模雲端業者 → 企業資料中心 → 一般市場
5. **台灣機會**：矽光子 PIC（眾達、華星光）+ 被動元件供應鏈

**對於投資者**：LPO 是「橋樑技術」，投資窗口有限但確定性較高。優先關注已切入 LPO 供應鏈的廠商，而非僅有潛力但尚未進入的。
